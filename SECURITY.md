# Security Policy

## Supported versions

Security updates are currently provided for the `main` branch.

## Reporting a vulnerability

Please report security vulnerabilities privately and do not open a public issue.

Include:

- A clear description of the vulnerability
- Reproduction steps or proof of concept
- Impact assessment
- Suggested mitigation (if available)

Response targets:

- Initial acknowledgement: within 3 business days
- Status update: within 7 business days
- Fix or mitigation plan: as soon as impact and scope are confirmed

## Secret management

- Never commit API secrets, signing keys, or credential files.
- Keep Firebase and environment-specific files local.
- Use restricted API keys and least-privilege Firebase rules.
